# 缺陷报告：底部导航标签大多无响应（WebView）

| 字段 | 值 |
|---|---|
| **产品** | 通信工程师考试 |
| **包名** | `com.csai.tongxin` **v1.0.1** |
| **严重程度** | 高（主导航对自动化失效，对用户可能不稳定） |
| **组件** | UNI-APP WebView 内底部标签栏 |

## 摘要
首页上，**滚动/甩动有效**，但底部标签 **我的 / 课程 / 学习** 的点击目标经密集 Y 扫描（`y=2560..2820`，步长 30）使用 `uitest uiInput click` 与 `hmdriver2 Driver.click` 均**无 UI 变化**。

**会员** 在 `(400, 2560)` 曾打开应用内登录（可能点中会员标签或附近控件）。其他标签始终无响应。

## 步骤
1. 进入首页（游客）。
2. 点击底部标签：会员 / 课程 / 学习 / 我的（图标 + 文案区域）。
3. 观察内容/选中标签是否变化。

## 预期
标签内容切换；选中标签指示器移动。

## 实际（实验室）
- 内容区 `uitest uiInput swipe/fling`：**有效**（截图差异 >15）。
- 多坐标标签点击：我的/课程/学习 **无变化**。
- `hmdriver2` 坐标点击同样如此。

## 为何重要
- 破坏核心信息架构（无法稳定进入 学习 / 我的）。
- 阻塞无选择器自动化；WebView 触控异常用户可能遇到同样死点击。

## 相关
Web 无障碍树为空 → 无法用 dumpLayout 确认点击目标；仅靠截图 + 包名判定。
